# 登录问题修复报告

**问题时间**: 2026-07-19 04:15  
**问题描述**: 管理端登录后没有任何反应，也没有跳转修改默认密码  
**状态**: ✅ 已修复

---

## 问题诊断

### 原因 1：admin 用户 first_login 状态错误
- **现象**: admin 用户 `first_login=0`
- **影响**: 后端返回 `passwordChangeRequired=false`
- **根因**: 之前测试时已改过密，`first_login` 被设置为 0

### 原因 2：auth store 缺少 isAuthenticated
- **现象**: `useAdminAuthStore` 未导出 `isAuthenticated` 计算属性
- **影响**: 路由守卫可能无法正确判断登录状态
- **根因**: 实现时遗漏了该属性

---

## 修复方案

### 修复 1：重置 admin 用户状态

```sql
UPDATE users SET first_login = 1 WHERE username = 'admin';
```

**验证**：
```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'

# 返回：
# "passwordChangeRequired":true ✅
```

### 修复 2：添加 isAuthenticated 计算属性

```typescript
// frontend-admin/src/store/auth.ts
import { ref, computed } from 'vue'; // 添加 computed

export const useAdminAuthStore = defineStore('adminAuth', () => {
  const token = ref(localStorage.getItem('admin_token') || '');
  const user = ref<AdminUser | null>(
    JSON.parse(localStorage.getItem('admin_user') || 'null'),
  );

  const isAuthenticated = computed(() => !!token.value); // 新增

  // ...

  return { 
    token, 
    user, 
    isAuthenticated,  // 导出
    login, 
    changePassword, 
    logout 
  };
});
```
