   ```typescript

     @PrimaryColumn() id: number = 1; // 单例配置
     @Column({ nullable: true }) primaryModelId: number;
     @Column({ type: 'json', nullable: true }) backupModelIds: number[]; // 最多 5 个

   ```

2. **迁移文件** `1700000070000-CreateOcrConfig.ts`：
   ```typescript
   await queryRunner.query(`
     CREATE TABLE ocr_config (
       id INT PRIMARY KEY DEFAULT 1,
       primaryModelId INT NULL,

   await queryRunner.query(`INSERT INTO ocr_config (id) VALUES (1)`);
   ```

   - `recognizeImage()` 改为按 primary → backup[0] → backup[1] ... 顺序重试
   - 每个模型调用失败后记录日志，继续下一个


**前端改动**：

### 问题 2：管理员初始密码登录后未强制改密

- **后端**：`AuthService.finalizeLogin()` 返回 `passwordChangeRequired: user.firstLogin === 1`
- **前端**：登录响应后未检查 `passwordChangeRequired` 字段，直接跳转 `/dashboard`

#### 根因
- 前端逻辑缺失：未处理 `passwordChangeRequired=true` 的情况

#### 整改方案

**前端改动**：

1. **`LoginPage.vue` 登录成功处理逻辑**：
   ```typescript
   const res = await http.post('/auth/login', loginForm);
   if (res.passwordChangeRequired) {
     ElMessageBox.alert(
       '检测到您使用的是初始密码，为保障账号安全，请立即修改密码。',
       '强制改密',
       {
         confirmButtonText: '立即修改',
         showClose: false,
         closeOnClickModal: false,
         closeOnPressEscape: false,

     ).then(() => {
       router.push('/change-password?force=1');
     });

     router.push('/dashboard');
   }
   ```

2. **新增 `/change-password` 页面**（如不存在）：

   - 检查 localStorage 中是否存在 `forceChangePassword` 标记
   - 若存在且当前路由不是 `/change-password`，强制重定向

**后端改动**：

### 问题 3：值班面板永远离线，点击重连无反应

#### 现状分析
- **前端**：`DutyDashboardPage.vue` 使用 `socket.io-client` 连接 `/duty` 命名空间
- **后端**：项目中**不存在** `/backend/src/modules/duty` 模块
- **根因**：后端未实现值班面板 WebSocket 服务

#### 根因
- **功能缺失**：值班面板 WebSocket 后端完全未实现

#### 整改方案

**后端改动**：

1. **新建模块** `backend/src/modules/duty`：
   ```
   duty/
   ├── duty.module.ts
   ├── duty.gateway.ts     // Socket.IO Gateway
   ├── duty.service.ts
   └── duty.controller.ts  // REST fallback: GET /duty/overview
   ```

2. **`DutyGateway` 实现**：
   ```typescript
   @WebSocketGateway({ namespace: '/duty', cors: { origin: '*' } })
   export class DutyGateway {
     @SubscribeMessage('join')
     handleJoin(client: Socket) {
       client.join('duty-room');
       this.emitOverview(client);
     }
   
     async emitOverview(client: Socket) {
       const data = await this.dutyService.getOverview();
       client.emit('overview', data);
     }
   }
   ```

3. **`DutyService.getOverview()` 实现**：
   - 查询近 24h 事件数（`MonitorEventEntity`）
   - 统计活跃告警数（`AlertEntity`）
   - 统计平台分布、情感分布、热点关键词
   - 返回结构与前端 `DutyDashboardPage` 期望一致

4. **定时推送**：
   - 使用 `@Cron('*/30 * * * * *')` 每 30 秒向所有连接客户端推送最新 overview

**前端改动**：

5. **`DutyDashboardPage.vue` 连接 URL 修正**：
   - 确认连接到正确的后端地址（当前可能硬编码了错误端口）
   - 增加连接失败时的用户友好提示："值班面板服务未启动，请联系管理员"

6. **`reconnect()` 函数完善**：
   ```typescript
   function reconnect() {
     if (reconnecting.value) return;
     reconnecting.value = true;
     socket.disconnect();
     socket.connect();
     setTimeout(() => {
       if (!wsConnected.value) {
         ElMessage.error('重连失败，值班面板服务可能未启动');
       }
       reconnecting.value = false;
     }, 5000);
   }
   ```

**验收标准**：
- [ ] 进入值班面板，连接状态显示「实时」
- [ ] 24h 事件数、活跃告警等数据正常显示
- [ ] 模拟断网后点击「重新连接」，5 秒内恢复「实时」状态
- [ ] 后端未启动时，显示友好错误提示

### 问题 6：系统日志未记录值班面板重连等用户操作

#### 现状分析
- 已有 `@AuditLog` 装饰器和 `AuditService`
- 但前端用户交互（如点击「重连」按钮）未调用后端审计接口

#### 根因
- **设计缺陷**：纯前端操作未主动上报到后端审计日志

#### 整改方案

**后端改动**：

1. **新增审计端点** `POST /api/admin/audit-events/record`：
   ```typescript
   @Post('record')
   @UseGuards(JwtAuthGuard)
   async recordFrontendAction(@Body() dto: FrontendAuditDto, @Req() req) {
     await this.auditService.record({
       actorId: req.user.sub,
       actorType: req.user.role === 'admin' ? 'admin' : 'user',
       module: dto.module,
       action: dto.action,
       resourceType: dto.resourceType || null,
       resourceId: dto.resourceId || null,
       title: dto.title,
       metadata: dto.metadata || {},
     });
     return { ok: true };
   }
   ```

2. **DTO 定义**：
   ```typescript
   export class FrontendAuditDto {
     module: string;
     action: string;
     resourceType?: string;
     resourceId?: number;
     title: string;
     metadata?: Record<string, any>;
   }
   ```

**前端改动**：

3. **封装审计工具函数** `utils/audit.ts`：
   ```typescript
   export async function recordAudit(params: {
     module: string; action: string; title: string;
     resourceType?: string; resourceId?: number; metadata?: any;
   }) {
     try {
       await http.post('/admin/audit-events/record', params);
     } catch { /* 审计失败不阻塞主流程 */ }

   ```

4. **在关键用户操作处调用**：
   - `DutyDashboardPage.vue`：
     ```typescript
     function reconnect() {
       recordAudit({ module: 'duty_dashboard', action: 'reconnect', title: '值班面板重新连接' });
       // ... 原有重连逻辑
     }
     ```
   - 其他页面类似：模板使用、配置保存、数据导出等

**验收标准**：
- [ ] 值班面板点击「重新连接」后，系统日志中出现「值班面板重新连接」记录

### 问题 7：TTS API Key 保存后重新登录消失

#### 现状分析
- **前端**：`TtsConfigPage.vue` 保存时调用 `POST /admin/config/tts`
- **后端**：`AdminController.setTtsConfig()` 使用 `@AuditLog` 但未检查 API Key 是否正确加密保存

#### 根因定位

需检查：
1. **后端是否正确加密存储**：
   - TTS 配置存储在哪个实体？（`TtsConfigEntity` / `AliyunConfigEntity` / JSON 字段？）
   - 是否使用 `CryptoUtil.encrypt()` 加密 API Key？

2. **前端是否正确回显**：
   - `GET /admin/config/tts` 是否返回解密后的 Key？
   - 前端 `onMounted` 是否正确加载配置到表单？

#### 整改方案（假设后端未加密存储）

**后端改动**：

1. **确认 TTS 配置实体**（如 `TtsConfigEntity`）：
   ```typescript
   @Entity('tts_config')
   export class TtsConfigEntity {
     @PrimaryColumn() id: number = 1;
     @Column() activeProvider: string;
     @Column({ type: 'text', nullable: true }) minimaxApiKeyEnc: string; // 加密存储
     @Column({ nullable: true }) minimaxGroupId: string;
     @Column({ type: 'text', nullable: true }) xiaomiApiKeyEnc: string; // 加密存储
   }
   ```

2. **`AdminService.setTtsConfig()` 加密逻辑**：
   ```typescript
   async setTtsConfig(dto: TtsConfigDto): Promise<void> {
     const config = await this.ttsConfigRepo.findOne({ where: { id: 1 } }) || new TtsConfigEntity();
     config.activeProvider = dto.activeProvider;
     if (dto.minimax?.apiKey) {
       config.minimaxApiKeyEnc = CryptoUtil.encrypt(dto.minimax.apiKey);
     }
     if (dto.xiaomi?.apiKey) {
       config.xiaomiApiKeyEnc = CryptoUtil.encrypt(dto.xiaomi.apiKey);
     }
     config.minimaxGroupId = dto.minimax?.groupId;
     await this.ttsConfigRepo.save(config);
   }
   ```

3. **`AdminService.getTtsConfig()` 解密逻辑**：
   ```typescript
   async getTtsConfig(): Promise<TtsConfigResponseDto> {
     const config = await this.ttsConfigRepo.findOne({ where: { id: 1 } });
     return {
       activeProvider: config?.activeProvider || 'minimax',
       minimax: {
         apiKey: config?.minimaxApiKeyEnc ? CryptoUtil.decrypt(config.minimaxApiKeyEnc) : '',
         groupId: config?.minimaxGroupId || '',
       },
       xiaomi: {
         apiKey: config?.xiaomiApiKeyEnc ? CryptoUtil.decrypt(config.xiaomiApiKeyEnc) : '',
       },
     };
   }
   ```

**前端改动**：

4. **`TtsConfigPage.vue` 确认加载逻辑**：
   ```typescript
   async function loadConfig() {
     const res = await http.get('/admin/config/tts');
     form.value.activeProvider = res.activeProvider;
     configs.value.minimax.apiKey = res.minimax?.apiKey || '';
     configs.value.minimax.groupId = res.minimax?.groupId || '';
     configs.value.xiaomi.apiKey = res.xiaomi?.apiKey || '';
   }
   onMounted(() => loadConfig());
   ```

**验收标准**：
- [ ] 保存 MiniMax API Key 后，刷新页面，Key 仍然显示（脱敏或完整）
- [ ] 重新登录后，TTS 配置页 Key 仍然存在
- [ ] 数据库中 `minimaxApiKeyEnc` 字段为加密密文
