# 全网舆情监测系统 - 需求文档

## 简介

构建一个覆盖微信公众号、视频号、抖音、小红书、快手、微博、百家号等主流平台的全网舆情监测系统。系统采用管理端与用户端分离架构，通过混合数据采集模式（聚合API为主 + 官方接口补充 + Webhook自推），实现关键词订阅、实时舆情监控、Webhook告警推送及实时大屏展示。

## 术语表

| 术语 | 定义 |
|------|------|
| 超级管理员 | 系统最高权限者，默认账号 admin，拥有配置管理和用户管理权限 |
| 普通用户 | 注册并通过手机号三要素认证的系统使用者，享有舆情监控订阅能力 |
| Webhook 机器人 | 用户创建的可接收舆情推送的外部回调地址，支持企业微信/钉钉/飞书/自定义格式 |
| 关键词 | 用户设置的用于匹配舆情内容的词或短语，支持精确匹配和模糊匹配 |
| 舆情事件 | 各平台公开发布的符合用户订阅规则的文本内容及其元数据（作者、阅读量、时间等） |
| 三要素认证 | 阿里云手机号三要素详细版 API，验证姓名+身份证号+手机号三者一致 |
| 数据采集适配器 | 系统对接各数据源的插件化模块，统一输出标准化的舆情事件结构 |

## 需求

### 需求模块 1：系统用户认证与权限

**用户故事：** AS 任何角色，I want 安全可靠的认证系统，SO THAT 不同角色按权限使用系统功能。

#### 验收标准

1. WHEN 用户访问系统登录页面，系统 SHALL 要求输入账号和密码，并在 5 次失败尝试后锁定 15 分钟
2. WHEN 超级管理员首次使用默认账号 admin 和密码 123456 登录，系统 SHALL 强制跳转到修改密码页面，完成修改后始可访问管理后台
3. WHEN 用户提交注册请求，系统 SHALL 要求提供手机号并通过阿里云短信验证码验证
4. WHEN 用户输入阿里云短信验证码，系统 SHALL 在 5 分钟内验证有效，超时则失效并要求重新获取
5. WHEN 用户完成手机号验证后注册，系统 SHALL 跳转到三要素认证页面，三要素通过后用户状态置为"已认证"
6. IF 用户密码连续 5 次输入错误，系统 SHALL 锁定账号 15 分钟，并发送短信通知
7. WHEN 用户点击"忘记密码"，系统 SHALL 通过阿里云短信发送重置链接，30 分钟内有效
8. WHILE 用户处于登录状态且 30 分钟无操作，系统 SHALL 自动登出会话

### 需求模块 2：管理端 - 系统配置管理

**用户故事：** AS 超级管理员，I want 管理阿里云短信和三要素认证的配置，SO THAT 认证和通知功能正常运行。

#### 验收标准

1. WHEN 超级管理员进入"阿里云配置"页面，系统 SHALL 显示短信服务配置区域（AccessKey、SecretKey、签名、模板代码）和三要素配置区域（产品 code）
2. WHEN 超级管理员修改并保存阿里云配置，系统 SHALL 发送测试短信验证配置有效性，配置成功后加密存储
3. WHEN 系统需要发送短信，系统 SHALL 使用管理端配置的阿里云凭据，不允许将凭据硬编码在代码中
4. WHEN 系统接口调用的阿里云凭据失效返回 4xx，系统 SHALL 记录错误日志并通过管理后台消息中心通知超级管理员
5. WHEN 超级管理员更新配置后，系统 SHALL 在 10 秒内生效，无需重启服务

### 需求模块 3：管理端 - 用户管理

**用户故事：** AS 超级管理员，I want 管理系统用户，SO THAT 控制平台访问和数据安全。

#### 验收标准

1. WHEN 超级管理员进入用户管理页面，系统 SHALL 以分页列表展示所有用户，每页 20 条，包含注册时间、认证状态、最后登录时间、账号状态
2. WHEN 超级管理员搜索用户，系统 SHALL 支持按用户名、手机号、注册时间段筛选
3. WHEN 超级管理员点击"封禁"用户，系统 SHALL 将该用户状态置为"已封禁"，该用户无法登录和访问所有功能
4. WHEN 超级管理员点击"解封"用户，系统 SHALL 将该用户状态恢复为"已认证"或"未认证"，用户可正常登录
5. WHEN 封禁/解封操作完成，系统 SHALL 通过阿里云短信通知被操作用户

### 需求模块 4：用户端 - 关键词订阅与舆情监控

**用户故事：** AS 普通用户，I want 设置关键词监控舆情，SO THAT 及时掌握特定话题在各平台的传播动态。

#### 验收标准

1. WHEN 用户创建关键词监控任务，系统 SHALL 要求用户输入关键词（支持精确词和通配符模式）、选择监测平台（多选）、设置监测频率（5/15/30/60 分钟）
2. WHEN 用户保存监控任务，系统 SHALL 立即执行一次初始数据拉取并在 30 秒内返回已有匹配结果
3. WHEN 用户查看监控任务列表，系统 SHALL 按最后更新时间和命中数倒序展示所有任务
4. WHEN 用户点击某个监控任务，系统 SHALL 展示该任务最近 100 条舆情事件列表，每条包含平台、标题、摘要、作者、发布时间、阅读/点赞/评论数
5. WHILE 监控任务处于启用状态，系统 SHALL 按照用户设置的频率自动从各平台适配器拉取数据并与关键词匹配
6. IF 某平台数据源 API 连续 3 次调用失败，系统 SHALL 暂停该平台的监控并在用户端标记"数据异常"
7. WHEN 用户删除监控任务，系统 SHALL 停止该任务的所有数据拉取和告警推送

### 需求模块 5：用户端 - Webhook 机器人

**用户故事：** AS 普通用户，I want 创建 Webhook 机器人接收舆情告警，SO THAT 实时获得监控结果推送。

#### 验收标准

1. WHEN 用户创建 Webhook 机器人，系统 SHALL 要求填写机器人名称、请求 URL、推送格式（企业微信/钉钉/飞书/自定义 JSON）、签名密钥（可选）
2. WHEN 用户测试 Webhook 机器人，系统 SHALL 发送一条测试消息到配置 URL，并在 5 秒内返回连通性结果
3. WHEN 用户将监控任务与 Webhook 机器人关联，系统 SHALL 在每次监控任务匹配到新舆情时，立即发送告警 payload 到所有关联的 Webhook URL
4. WHEN Webhook 推送返回 4xx/5xx 状态码，系统 SHALL 实施指数退避重试（最多 3 次，间隔 5/15/45 秒），3 次均失败则在用户端标记为"推送异常"
5. WHEN 用户启用"定时推送"模式，系统 SHALL 按用户设置的推送时间（每日/每 6 小时/每小时）将汇总报告推送到 Webhook 机器人
6. WHEN 用户编辑 Webhook 机器人配置，系统 SHALL 在保存后对所有关联任务的下一次推送生效，不影响进行中的推送

### 需求模块 6：用户端 - 实时舆情大屏

**用户故事：** AS 普通用户，I want 通过实时大屏直观查看舆情动态，SO THAT 快速掌握关键信息变化趋势。

#### 验收标准

1. WHEN 用户进入实时大屏页面，系统 SHALL 通过 WebSocket 建立持久连接，初始加载该用户所有监控任务的最新舆情数据
2. WHILE 用户停留在实时大屏页面，系统 SHALL 通过 WebSocket 实时推送新增的匹配舆情事件，端到端延迟不超过 3 秒
3. WHEN 用户在大屏上点击某条舆情事件，系统 SHALL 在新窗口打开该内容的原始链接
4. WHEN 用户在大屏上选择平台筛选器，系统 SHALL 仅展示选定平台的舆情事件和数据统计
5. WHEN 用户在大屏上点击关键词筛选，系统 SHALL 仅展示该关键词相关的舆情事件和数据统计
6. WHILE 大屏处于活跃状态，系统 SHALL 每 5 秒更新一次数据统计看板（总舆情数、各平台占比、情感分布、趋势曲线）

### 需求模块 7：用户端 - 阿里云手机三要素认证

**用户故事：** AS 普通用户，I want 通过手机号三要素认证，SO THAT 完成账号实名认证获得完整功能权限。

#### 验收标准

1. WHEN 用户完成注册后首次登录，系统 SHALL 跳转到三要素认证页面，要求输入真实姓名、身份证号、手机号
2. WHEN 用户提交三要素认证信息，系统 SHALL 调用阿里云手机号三要素详细版 API 进行校验
3. IF 三要素认证通过，系统 SHALL 将用户状态更新为"已认证"，用户始可创建监控任务和使用 Webhook 功能
4. IF 三要素认证失败（姓名/身份证/手机号不匹配），系统 SHALL 提示具体哪个要素不匹配，用户在 24 小时内可重试 3 次，超限后锁定 7 天
5. IF 三要素认证接口返回服务不可用，系统 SHALL 提示用户稍后重试并记录日志

### 需求模块 8：数据采集适配器

**用户故事：** AS 系统，I want 对接多平台数据源的适配器层，SO THAT 统一采集和标准化舆情数据。

#### 验收标准

1. WHEN 数据采集适配器启动，系统 SHALL 根据管理端配置的数据源信息初始化各平台适配器实例
2. WHEN 适配器从各平台获取到原始数据，系统 SHALL 统一转换为标准舆情事件格式（platform、title、content、author、publishTime、url、readCount、likeCount、commentCount、sentiment）
3. WHEN 适配器调用外部 API 超时（超过 10 秒），系统 SHALL 记录超时日志并返回空结果，不影响其他平台的数据采集
4. WHEN 用户通过 Webhook 方式推入舆情数据，系统 SHALL 验证 Webhook 签名后将其格式化入库，与其他平台数据同等处理
5. WHEN 系统启动定时采集任务，系统 SHALL 根据用户设置的任务频率将任务分发至各平台适配器并行执行

### 需求模块 9：阿里云短信通知

**用户故事：** AS 系统，I want 在关键事件节点通过短信通知用户，SO THAT 用户及时获知账号和安全状态变化。

#### 验收标准

1. WHEN 用户请求发送登录验证码，系统 SHALL 调用阿里云短信 API 发送 6 位纯数字验证码至用户手机号
2. WHEN 用户请求发送注册验证码，系统 SHALL 调用阿里云短信 API 发送 6 位纯数字验证码至用户手机号
3. WHEN 用户账号被管理员封禁或解封，系统 SHALL 调用阿里云短信 API 发送账号状态变更通知
4. WHEN 用户设置的监控关键词匹配到涉及人身安全/重大负面等敏感内容，系统 SHALL 通过阿里云短信发送紧急通知（用户需在 Webhook 规则中开启此选项）
5. WHEN 阿里云短信单日发送量超过管理端设置的日发送上限，系统 SHALL 停止发送并告警管理员

### 需求模块 10：权限与数据隔离

**用户故事：** AS 系统，I want 确保用户间数据完全隔离，SO THAT 每个用户只看到属于自己的监控数据和配置。

#### 验收标准

1. WHEN 用户 A 创建任何监控任务、Webhook 机器人和舆情数据，系统 SHALL 确保用户 B 在任何接口和页面都不可见用户 A 的任何数据
2. WHEN 用户注销账号，系统 SHALL 在 72 小时后清理该用户的所有监控任务、Webhook 配置和关联的舆情数据
3. WHEN API 请求不携带有效认证 Token，系统 SHALL 返回 401 状态码和统一错误格式
4. WHEN API 请求携带过期 Token，系统 SHALL 返回 401 状态码，客户端应在收到 401 后自动跳转至登录页
5. WHEN API 请求携带有效 Token 但当前用户已被封禁，系统 SHALL 返回 403 状态码和封禁原因
