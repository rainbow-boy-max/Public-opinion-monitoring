## 三、优化升级工作包

### 🔴 P0 — 现有功能缺陷修复（必须立刻修复）

| ID | 问题 | 影响范围 | 工作量 |
|---|---|---|---|
| F-01 | webhook push 通道对接适配器校验 | 用户端 Webhook 可能失效 | 1h |
| F-02 | collector adapter fetch 全部超时（无 fallback） | 所有平台采集为 0 | 2h |
| F-03 | migration: user table 的 `role` ENUM 改为 varchar（与 entity 对齐） | 数据库迁移兼容 | 1h |
| F-04 | frontend-user 登录后 redirect 逻辑缺失 | 用户登录后空白 | 1h |
| F-05 | export report 缺失（pr-reports 页面无导出入口） | 无法导出舆情报告 | 2h |
| F-06 | RealtimeScreenPage SSE 流未对接真实数据 | 大屏始终空数据 | 3h |

## 五、关键决策点

| # | 决策 | 建议 | 说明 |
|---|---|---|---|
| 1 | P0 修复 → P1 → P2 → P3 顺序是否同意？ | ✅ 同意 | 先修缺陷，再补齐竞品能力，最后增值 |

| 5 | P3-05 多租户需要重新设计 auth guard | 另起专项 | 涉及用户表改造、数据隔离、RBAC，本期不做 |

## 7 个待你点头的决策

1. **Phase 1 先做 P0 修复（6 项）＋ P1-03(预警)＋P1-08(大屏)** — 这些阻塞用户直接使用
2. **情感分析增强（P1-07）是否用 LLM 兜底** — 建议用现有 `KbScoringService.scoreDocument` 路径复用
3. **热点话题发现（P1-06）用滑动窗口还是固定周期** — 建议 1h 滑动窗口
4. **微信小程序延后到 Phase 3**
5. **传播路径（P1-01）先走 ECharts 关系图快速上线**
6. **知识库批量上传（P1-09）增加多文件拖拽支持**
7. **短视频监测只做抖音+快手，不做全平台**

确认后按 Phase 1 开始施工。
