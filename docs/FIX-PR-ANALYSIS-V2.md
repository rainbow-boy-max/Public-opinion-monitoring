# 自定义舆情分析功能全面设计方案

## 竞品对标参考

| 竞品 | 输入方式 | 处理流程 | 输出 |
|------|---------|---------|------|
| 新浪舆情通 V助手 | 一句话描述/事件链接/已有事件 | 4 Agent 协同（监测→分析→追踪→报告）| 5 维度报告：概况/数据/观点/媒体/建议 |
| 清博舆情 | 自定义规则+分析模板 | 大模型精调+知识图谱 | 情感分析/传播路径/应对策略 |
| 识微商情 | 事件文本/竞品关键词 | AI 大模型从收集到洞察 | 负面提炼/传播逻辑/品牌策略 |

## 功能设计

### 三种输入方式

用户通过 Tab 切换选择输入方式：

```
┌────────────────────────────────────────────────┐
│  [ 选择事件 ]  [ 输入链接 ]  [ 上传文档 ]       │
├────────────────────────────────────────────────┤
│                                                  │
│  Tab1: 选择事件                                   │
│  ┌──────────────────────────────────────────┐   │
│  │ 搜索已有舆情事件...                       │   │
│  ├──────────────────────────────────────────┤   │
│  │ ○ [#1523] 品牌A产品质量问题 (微博·23分钟前) │   │
│  │ ● [#1521] 新品发布会负面反馈 (微信·1h前)   │   │
│  │ ○ [#1518] 竞品B价格调整 (抖音·3h前)        │   │
│  └──────────────────────────────────────────┘   │
│                                                  │
│  Tab2: 输入链接                                   │
│  ┌──────────────────────────────────────────┐   │
│  │ 输入网页链接，例如: https://...            │   │
│  └──────────────────────────────────────────┘   │
│                                                  │
│  Tab3: 上传文档                                   │
│  ┌──────────────────────────────────────────┐   │
│  │ [📎 上传文件] (支持 PDF/Word/图片/TXT)     │   │
│  │ 已上传: 舆情截图.png (2.3MB)              │   │
│  └──────────────────────────────────────────┘   │
│                                                  │
│  AI 智能体 (可选)                                 │
│  └── [选择AI智能体 ▼] ───────────────────┘      │
│                                                  │
│  [ 提交分析 ]                                     │
└────────────────────────────────────────────────┘
```

### 处理流程

```
输入 → 内容采集 → 事件摘要 → 智能体分析 → 报告生成
```

| 步骤 | 说明 | 后端实现 |
|------|------|---------|
| 1. 输入处理 | 根据 Tab 类型获取分析内容 | `POST /pr/analyze` 统一入口 |
| 2. 内容采集 | 链接→爬取正文；文档→OCR/解析；事件→数据库 | 新增 `urlFetch` + 复用 `upload` |
| 3. 事件摘要 | LLM 总结分析内容 | `startReportFromText` |
| 4. 智能体分析 | 选择 AI 智能体或默认，生成应对方案 | 复用 `agentsService` |
| 5. 报告生成 | 保存报告并返回结果 | 现有流程 |

### 后端新增

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /pr/analyze` | 已存在 | body: `{ eventId \| url \| text,?agentId }` |
| `GET /pr/reports/:id` | 已存在 | 查询报告状态 |
| `POST /util/fetch-url` | **新增** | 抓取网页链接内容返回文本 |
| `POST /util/ocr-document` | **新增** | OCR 识别上传文档中的文字 |
