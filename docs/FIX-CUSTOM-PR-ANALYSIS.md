# 自定义舆情分析功能设计方案

## 竞品调研结论

| 竞品 | 自定义分析功能 | 核心能力 |
|------|--------------|---------|
| 新浪舆情通 V助手 | 多智能体协同，一句话创建监测方案，8步→2步 | 事件识别/风险等级/传播追踪 |
| 清博舆情 | 自定义规则与分析模板，多语言翻译 | 报告模板灵活定制 |
| 识微商情 | AI大模型分析，从"信息收集"到"深度洞察" | 情感分析/传播路径/应对建议 |
| TOOM舆情 | 双活LLM+行业精调蒸馏模型 | 情感识别92%/30秒预警 |

## 功能设计

### 用户端

在现有 PR 报告列表上增加"自定义舆情分析"功能入口，点击后弹出对话框：

```
┌─────────────────────────────────────────────┐
│  自定义舆情分析                               │
├─────────────────────────────────────────────┤
│  分析内容 (必填)                              │
│  ┌─────────────────────────────────────────┐│
│  │ 输入舆情事件描述、新闻链接、或粘贴文本... ││
│  │                                         ││
│  │  (textarea, 5行)                        ││
│  └─────────────────────────────────────────┘│
│                                             │
│  AI 智能体 (可选)                            │
│  └── [选择AI智能体 ▼] ──────────────────┘  │
│                                             │
│  [提交分析]          [取消]                  │
└─────────────────────────────────────────────┘
```

提交后：
1. 调用 `POST /api/pr/reports/from-text` 提交分析
2. 返回报告 ID，显示"分析中..."进度状态
3. 轮询 `GET /api/pr/reports/:id` 等待完成
4. 完成后自动打开报告详情对话框
5. 支持语音播报、导出

### 管理端

管理端增加"AI 公关分析"页面，功能与用户端一致，额外增加：
- 查看所有用户提交的分析报告列表
- 标记报告为"已读/未读"
- 删除报告
- 按用户、时间筛选

### 后端复用

后端已存在完整支持：
- `POST /api/pr/reports/from-text` — 自定义文本分析
- `POST /api/pr/reports/:id` — 查询报告状态
- `GET /api/pr/reports` — 列表
- LLM 模型自动选择已启用模型
