# 6 项问题整改方案

## 问题 1：管理端热点话题页面无内容

**根因**：页面仅显示监控任务热点（`GET /api/hot-topics`），无事件数据时为空。平台热点数据（`GET /api/hot-topics/platform`，9 平台 258 条实时数据）未使用。

**修复**：重写为双 Tab 结构：
- Tab1「平台热点」：展示 9 大平台实时榜单（微博/抖音/百度/知乎/B站/头条/虎扑/36氪/GitHub）
- Tab2「任务热点」：保留现有监控任务热点逻辑

## 问题 2：管理端无工单处理系统

**根因**：路由 `/work-orders` 和页面 `WorkOrdersPage.vue` 已存在（新建/回复/完结/评分按钮齐全），但菜单位置不突出。

**修复**：确认菜单路径正确，在 AdminLayout 中将工单菜单移至显眼位置。添加空状态引导文案。

## 问题 3：知识图谱 LLM 主/备模型配置

**新增**：`KnowledgeGraphConfigPage.vue`
- 下拉选择主模型（来自 LLM 模型列表）
- 多选最多 5 个备用模型
- 后端 `llm_models` 表新增 `is_kg_primary` / `is_kg_fallback` 字段
- 管理端/用户端知识图谱页面显示当前主+备模型标签

## 问题 4：热点话题显示缺少标题标签

**根因**：热点卡片中的数字和进度条没有说明性标题，用户看不懂含义。

**修复**：在每个热点卡片中明确标注：
```
排名 热度值(万)  平台标签  分类标签  趋势箭头
```

| 元素 | 说明标签 |
|------|---------|
| 排名数字 | #1 #2 ... |
| 热度值 | 热度柱状条 + "热度：XXX万" |
| 平台 | 彩色平台标签（微博红/抖音蓝等） |
| 分类 | 灰色分类标签（社会/科技/娱乐等） |
| 趋势 | 上升/下降箭头 + 百分比 |

## 问题 5：AI 公关无反应

**根因**：页面首次加载 `loadReports()` 返回空列表后显示空状态，用户需要手动点击"加载示例报告"。已修复为自动加载 mock 数据。

**修复确认**：`onMounted` 中加载报告完成后，若列表为空自动调用 `loadMockReport()` 填充示例数据。

## 问题 6：所有功能页面缺少引导文案

**需求**：每个功能页面需有清晰的引导文案，告诉用户该页面是做什么的、怎么用。

**新增统一引导组件**：`PageGuide.vue`

| 页面 | 引导文案 |
|------|---------|
| 概览 | 查看全网舆情态势总览，了解声量趋势、情感分布和最新事件 |
| 监控任务 | 创建监控任务后，系统将自动采集各大平台的相关舆情信息 |
| Webhook | 配置 Webhook 后，当有新的舆情事件时系统将自动推送通知 |
| 实时大屏 | 实时展示舆情数据大屏，支持自定义 widget 布局 |
| AI 公关 | 使用 AI 生成舆情分析报告、公关应对方案和声明话术 |
| 事件脉络 | 按时间轴查看舆情事件的发展脉络和关键节点 |
| 预警中心 | 设置预警规则，当舆情指标触发阈值时自动通知 |
| 竞品对比 | 创建竞品组后，可对比各竞品的声量、情感和平台分布 |
| 传播路径 | 查看舆情事件在不同平台间的传播扩散路径 |
| 多维对比 | 选择多个关键词组和时间范围进行对比分析 |
| 热点话题 | 查看各大平台实时热搜榜单和基于监控任务的热点发现 |
| 短视频监控 | 专项监控抖音/快手/小红书平台的短视频舆情 |
| 自定义面板 | 自由拖拽布局，打造专属舆情监控仪表盘 |
| 工单系统 | 提交工单后可由管理员进行人工研判和处置 |
| 知识图谱 | 从舆情事件中自动抽取实体和关系，构建知识图谱 |

在每个页面顶部（标题下方）添加一行引导文案，使用 `<p class="page-guide">` 样式，浅色副文本。
