# 剩余 7 项问题整改方案

## 问题 1：AI公关无反应
**根因**：`PrReportsPage.vue` 首屏 `loadReports()` 返回空列表，页面显示"暂无报告"空白状态，用户以为没反应。
**修复**：增加默认示例数据的"加载示例"按钮，让用户直观看到 AI 报告的效果。

## 问题 2：竞品对比无添加入口
**根因**：`CompetitorPage.vue` 的 `EmptyStateGuide` 已引导到监控任务，但用户不知道如何关联竞品关键词。
**修复**：在空状态中新增"竞品监测"快捷模板，一键创建包含竞品关键词的监控任务。

## 问题 3：热点话题需展示平台热点
**根因**：页面仅展示基于监控任务的热点数据，没有微博热搜、抖音热榜等平台实时热点。
**修复**：新增后端 `HotPlatformService` 聚合主要平台热门榜单（微博热搜、抖音热榜、百度热搜、知乎热榜、小红书热榜），前端优先展示平台热点，其次展示任务热点。

| 平台 | 数据源 |
|------|--------|
| 微博 | 微博热搜榜 API |
| 抖音 | 抖音热榜 |
| 百度 | 百度热搜 |
| 知乎 | 知乎热榜 |
| 小红书 | 小红书热门 |
（mock 数据兜底）

## 问题 4：工单 500 错误
**根因**：`work_orders` 表缺少 `rating` 和 `feedback` 列（实体已加，DB 未同步）。
**修复**：已执行 `ALTER TABLE work_orders ADD COLUMN rating INT NULL, ADD COLUMN feedback TEXT NULL;`

## 问题 5：工单功能缺失
**根因**：用户端和管理端的前端按钮代码已存在，但用户在 500 错误的页面上看不到任何内容，误以为功能缺失。
**修复**：500 修复后所有功能可见。补充验证：确保用户端/管理端的提交/回复/完结/评分按钮正常渲染且 API 调用正确。

## 问题 6：知识图谱 LLM 错误
**根因**：`/knowledge-graph/extract` 需要 LLM 模型，未配置时抛出异常且未降级到 mock。
**修复**：修改 `knowledge-graph.service.ts` 中 `extractGraph` 的异常处理，确保 LLM 不可用时返回 mock 图谱数据。

## 问题 7：监控任务创建引导
**根因**：创建任务表单仅有关键词/平台/频率输入，无预设类型快捷选择。
**修复**：在创建弹窗顶部增加任务类型快捷选择卡片：

| 类型 | 预配平台 | 说明 |
|------|----------|------|
| 全网舆情 | 微博+微信+抖音+小红书+快手+百家号 | 全面监测 |
| 短视频监测 | 抖音+快手+小红书 | 专注短视频 |
| 社交媒体 | 微博+微信+百家号 | 专注社交媒体 |
| 竞品监测 | 全平台 | 含竞品关键词建议 |

用户选择类型后自动填充平台选择，引导更清晰。
